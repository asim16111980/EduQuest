## Architecture

- **Path aliases**: Use `@/` prefix (e.g., `@/components/ui/Button`)
- **Stage colors**: Primary=blue/yellow, Preparatory=green/teal, Secondary=purple/indigo
- **Auth**: Currently simulated (localStorage-like mock). Swap in `supabase.auth` for production.
- **Admin Auth**: JWT-like tokens in `sessionStorage`, rate limiting, 30min session timeout, audit log
- **Admin pages**: Lazy-loaded via `React.lazy()` in `App.tsx` with `<Suspense>` + `AdminPageSkeleton` fallback
- **Testing**: Vitest + React Testing Library — 29 unit tests (see `src/test/`)
- **Game questions**: Procedurally generated by subject (math=arithmetic, arabic=Arabic grammar, english=vocab, etc.)
- **Progress tracking**: In-memory Zustand store (useProgressStore). Connect to Supabase `user_progress` table for production.
- **Admin data**: All in-memory mock. Every store/page has comments marking Supabase integration points.

## Database Schema (Supabase - for production)

```sql
-- users: id (uuid), name (text), email (text), grade_id (int), created_at (timestamptz), role (text: 'student'|'admin'), status (text: 'active'|'suspended')
-- grades: id (int), name_ar (text), name_en (text), stage (text), order (int), is_active (bool)
-- games: id (uuid), title (text), title_ar (text), description (text), description_ar (text), subject (text), grade_id (int), difficulty (text), thumbnail_url (text), is_visible (bool)
-- questions: id (uuid), game_id (uuid), text (text), options (jsonb), correct_answer (int), difficulty (text)
-- user_progress: id (uuid), user_id (uuid), game_id (uuid), score (int), stars (int), completed_at (timestamptz)
-- audit_log: id (uuid), action (text), target (text), details (text), created_at (timestamptz)
```

## File Header Comments

Every new file created must include a JSDoc comment at the very top describing what it does, its key exports, and any important notes.

### Format
```tsx
/**
 * @file ComponentName.tsx
 * @description Brief description of what this file does.
 * @exports ComponentName — what it is
 * @dependencies List any non-obvious dependencies
 * @notes Any important implementation notes or TODOs
 */
```

### Examples

**Page component:**
```tsx
/**
 * @file UsersPage.tsx
 * @description Admin page for managing platform users — view, edit, delete, and add students.
 * @exports UsersPage (default)
 * @dependencies adminStore, DataTable, Modal, ConfirmDialog
 * @notes All data is currently in-memory mock. Connect to Supabase `users` table for production.
 */
```

**Utility/lib file:**
```tsx
/**
 * @file adminAuth.ts
 * @description Secure admin auth utilities — JWT-like token generation, rate limiting, session timeout, audit logging.
 * @exports generateToken, verifyToken, checkRateLimit, recordFailedAttempt, logAction, getAuditLog
 * @notes Token is stored in sessionStorage only. Never use localStorage for admin tokens.
 */
```

> This rule applies to all new `.ts` and `.tsx` files. Existing files should be updated when significantly modified.
